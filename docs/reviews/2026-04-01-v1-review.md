## Critical Findings (Must Fix)

| ID | Finding | Remediation |
|----|---------|-------------|
| C1 | writeChain poisons on any I/O error *(main.ts:72-105)* | Add `.catch()` recovery on each `.then()` segment *(one disk error permanently deadlocks all audit operations for the session)* |
| C2 | Tab items are `<div>`, not keyboard accessible *(SettingsTab.ts:79-87)* | Use `<button>` with `role="tab"`, `aria-selected`, arrow-key nav *(violates WCAG 2.1.1 — tabs unreachable by keyboard)* |
| C3 | Picker list items are `<div>`, not keyboard accessible *(TagPickerModal.ts:83, VaultFolderModal.ts:49, ApiKeyTab.ts:223)* | Replace `createDiv` with `createEl('button')` or add `tabindex="0"` + keydown handlers *(violates WCAG 2.1.1)* |

### Code Examples

**[C1] writeChain poisons on I/O error**
```typescript
// Before — one failure breaks all future audit writes:
writeChain = writeChain.then(async () => {
  const existing = await adapter.read(logPath).catch(() => '');
  await adapter.write(logPath, existing + line);
});

// After — each segment recovers independently:
writeChain = writeChain
  .then(async () => {
    const existing = await adapter.read(logPath).catch(() => '');
    await adapter.write(logPath, existing + line);
  })
  .catch(err => kadoError('audit write failed', { error: String(err) }));
```

**[C2] Tab items are divs**
```typescript
// Before:
const tab = tabStrip.createDiv({ cls: 'kado-tab', text: label });

// After:
const tab = tabStrip.createEl('button', {
  cls: `kado-tab${id === this.activeTab ? ' is-active' : ''}`,
  text: label,
});
tab.setAttribute('role', 'tab');
tab.setAttribute('aria-selected', String(id === this.activeTab));
tab.setAttribute('aria-controls', 'kado-tab-content');
```
